01 · 推薦 Claude Desktop

為什麼我轉到 Claude Desktop

最主要的原因很單純：回到官方原生的 App——不用再自己維護一套第三方工具，功能和更新都跟著官方走，直觀又省心。
以前 · Better Agent Terminal

第三方工具「套殼」跑 Claude，更新要自己來——跑指令、去 GitHub 下載。

官方一有新功能，常常跟不上。

現在 · 官方原生 Claude Desktop

官方原生 App，功能第一時間就有。

更新自動跟上，穩定又不用操心。

04 · 日常應用

Connectors：連上你的其他服務

是什麼Desktop 內建 Connectors，直接連 Gmail、Google Drive、ClickUp… 不用手動複製貼上。
案例 · ClickUp 工單把工單貼給 Claude 分解、分析，再把結論「寫回」工單留言；同事後續直接依這則留言判斷、結案。
重點Connectors 不只「讀」，也能「寫回」——AI 協作變成團隊共享的紀錄，而不是聊完就消失。

06 · 日常應用

Routines：自動去 Discord 發提醒

看得見的自動化呼應前面的「一目瞭然」——你排了哪些自動化，全列在一張清單裡，開關與執行結果一眼掌握，不像傳統排程藏在背後。
案例 · 現場 demo排程 discord-drink-water-remind：到點 Claude 用 Claude in Chrome 開瀏覽器 → 進 Discord 發提醒。
真正的重點排程怎麼設？就照你平常怎麼用滑鼠操作那個網頁，描述一次即可——不用懂 API。公司自己的後台網站，也很適合這樣排程。

07 · 工作應用

介紹 Slot Game Builder

是什麼用 Claude 一步一步迭代開發出來的 Unity 編輯器工具，一鍵把 Slot 新專案的基礎架構自動建好。
重點資料夾結構、和 Server 對接的 Proto 檔案、美術資源、多語系、場景、Loading UI…這些專案基礎架構自動設置好，開發者不用再花時間搭建，能把時間留給真正重要的遊戲邏輯。

08 · 工作應用

新專案「一鍵初始化」

  • 實踐醞釀中的想法：「新專案自動初始化」，現在是 Slot Game Builder 裡一個真的能按的按鈕。
  • 一鍵套用所有設定：輸入專案名稱、按一個鈕，整套設置一次跑完（資料夾、Proto 檔、美術資源、字串表、場景、UI…）；每步都有勾選與進度條，失敗不擋後面。

08 · 工作應用

刷新 NAS 美術資源（單一資料夾）

痛點美術更新資源後，得自己連上 NAS 把新檔抓下來，再一個個取代掉專案裡的舊資源——手動又瑣碎。
怎麼用在 Art 底下對要更新的資料夾按右鍵 →「刷新 NAS 美術資源(資料夾)」。它自動對應到 NAS 上同一路徑，確認後把最新素材抓下來、覆蓋本地。
重點只更新你選的那個資料夾，不用整包重抓；美術一有更新，右鍵一下就同步。

遊戲換皮

愈用愈順這些之所以愈做愈快，是因為——在跟 Claude 一來一往、累積專案慣例的過程中，它會愈來愈懂你的風格、也愈懂這個專案，出手就愈精準。

10 · 接下來

Slot Game Plugin：把做遊戲的本事拆成一套 Skills

  • 一個 Plugins 資料夾，裝著一套 Skills：把「做一款 Slot Game」的每個環節，拆解成一個個 Skills。
  • 每個 Skill 就是一份「說明書」：把這個環節「怎麼做、怎麼檢查」寫清楚，AI 照著流程處理、照著標準檢查。
  • 每個人維護自己專精的那幾個 Skills，AI 就靠這套說明書組裝一款遊戲。
